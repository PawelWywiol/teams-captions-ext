# teams-captions-ext

Privacy-first browser extension that captures Microsoft Teams meeting captions, persists them locally, and runs on-demand analysis against a local LLM proxy you control.

- **No telemetry**, no cloud storage — transcripts live in your browser's IndexedDB.
- **Map-reduce summarisation** with per-chunk caching so re-runs over long meetings stay fast.
- **Custom prompts** per analysis; defaults are tuned for topic-aware meeting summaries.
- **Markdown output** rendered as raw text + one-click copy to clipboard.
- **Multi-session** model — every meeting URL is its own session, indefinitely retained until you delete it.

## Install

Each release on the [GitHub Releases page](https://github.com/PawelWywiol/teams-captions-ext/releases) publishes two artifacts:

| File                                            | Browser                                  | Notes                                                              |
| ----------------------------------------------- | ---------------------------------------- | ------------------------------------------------------------------ |
| `teams-captions-ext-chromium-<TAG>.zip`         | Chrome / Edge / Brave / Arc / Chromium   | See [`docs/install-chrome.md`](./docs/install-chrome.md).          |
| `teams-captions-ext-safari-unsigned-<TAG>.zip`  | Safari (macOS, developer mode)           | See [`docs/install-safari.md`](./docs/install-safari.md).          |

You also need an LLM proxy reachable at the URL you configure in **Settings**. The intended one is [`cli-llm-proxy`](https://github.com/PawelWywiol/cli-llm-proxy) (Fastify on `127.0.0.1:11434` with optional bearer auth, Claude/Copilot/Gemini adapters).

## Quick start (development)

```bash
pnpm install
pnpm build:extension          # bundles dist/extension/
pnpm package:chromium         # dist/teams-captions-ext-chromium.zip
pnpm package:safari           # macOS only — dist/teams-captions-ext-safari-unsigned.zip
```

Other useful scripts: `pnpm typecheck`, `pnpm lint`, `pnpm test`, `pnpm format:check`, `pnpm audit:deps`, `pnpm dev:mock-proxy`.

## How it works

```
Teams DOM → content/  → background/  → IndexedDB (Dexie)
                                    ↘
                                     LLM orchestrator
                                       chunker → hasher → cache → map calls → reduce
                                                                              ↓
                                                                     latest summary in DB
                                                                              ↑
                                  Sessions UI ← liveQuery ─────────────────────
```

Architecture deep dive: [`docs/architecture/`](./docs/architecture/).

## Privacy and security

- Transcripts and summaries never leave the machine until you click **Analyze**, and then only to the proxy you configured.
- The proxy host permission is requested at runtime (not in the static manifest) and can be revoked at any time.
- Bearer tokens live in `chrome.storage.session` (browser session lifetime), not in persistent storage.
- The system prompt explicitly instructs the model to treat the transcript as data, not as instructions, mitigating prompt injection from caption content.
- Threat model: [`docs/architecture/threat-model.md`](./docs/architecture/threat-model.md).

## Releases and CI

- `CI` (Ubuntu, Node 22 & 24) runs typecheck / lint / format / test / dep audit on every PR and push to `main`. Pull requests additionally upload a Chromium zip artifact for 14 days.
- `Release` (macOS) runs on every push to `main`, every `v*` tag, and on manual `workflow_dispatch`. It re-runs verification, builds both bundles, and attaches them to a GitHub Release with autogenerated notes:
  - **Main pushes** publish a **prerelease** tagged `v${pkg.version}-build.${run_number}` (one per merge).
  - **`v*` tags** publish a **stable** release.

To cut a stable release:

```bash
git tag v0.1.0
git push origin v0.1.0
```

Prereleases can also be promoted to stable in the Releases UI without rebuilding.

## Project layout

```text
src/
  background/    service worker + session orchestrator + message router
  content/       DOM observer for Teams captions
  popup/         action popup (Preact)
  options/       settings page (Preact)
  sessions/      full-page session browser (Preact)
  ui/shared/     design tokens + primitives + liveQuery hook
  shared/
    db/          Dexie schema + helpers (sessions, entries, chunks, summaries)
    llm/         chunker, hasher, prompts, payload, orchestrator
    storage.ts   chrome.storage policy for settings + bearer token
    types.ts     shared types and message protocol
  aliases/       participant alias resolver
  api/           proxy client + runtime-permission gate
  manifest.json  MV3 manifest

tests/           Vitest unit + integration tests (fake-indexeddb)
docs/            architecture + install docs
scripts/         build, packaging, mock proxy
```
